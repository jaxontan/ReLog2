## Project-specific rules

- Feature-first MVVM: `View → ViewModel → Repository → Service → Firebase SDK`.
- Files/dirs: `snake_case`. Classes: `UpperCamelCase`. vars/methods: `lowerCamelCase`.
- No `build()` > ~100 lines; use `const` constructors; lazy lists.
- State: Riverpod. Routing: `go_router`. Backend: Firebase
  (Auth, Firestore, Storage, Crashlytics).
- Definition of Done (PLAN.md §5): `flutter analyze` clean, `flutter test`
  green, every ViewModel + Repository has a test, Ponytail review passed.
